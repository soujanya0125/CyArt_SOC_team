# Task 2: Response Documentation

## Objective
Master the documentation of incident response procedures and findings.

## Documentation Standards

### Incident Response Report Template

#### Header
- **Incident ID**: [Unique identifier]
- **Date Reported**: [Date and time]
- **Reported By**: [Name and role]
- **Severity Level**: [Critical/High/Medium/Low]

#### Initial Assessment
- **Alert Title**: [Alert name]
- **Affected System(s)**: [List systems]
- **Initial Description**: [What happened]
- **Detection Time**: [When was it detected]
- **First Action Taken**: [Initial response]

#### Investigation Details
- **Timeline**: [Chronological list of events]
- **Evidence Collected**: [Files, logs, data]
- **System Access History**: [Login records, access logs]
- **Suspicious Activities**: [Detailed findings]

#### Response Actions
- **Containment Measures**: [Actions taken to stop spread]
- **Systems Isolated**: [List of isolated systems]
- **Accounts Disabled**: [User accounts affected]
- **Credentials Reset**: [Passwords/tokens changed]

#### Eradication Steps
- **Malware Removed**: [Cleanup actions]
- **Vulnerabilities Patched**: [Updates applied]
- **Configuration Changes**: [Security improvements]
- **Data Restored**: [Backup restoration]

#### Recovery and Monitoring
- **System Restoration**: [Return to normal operations]
- **User Communications**: [Notifications sent]
- **Monitoring Implemented**: [Continued observation]
- **Follow-up Actions**: [Future preventive measures]

#### Lessons Learned
- **Root Cause**: [Why did this happen]
- **Process Improvements**: [What should change]
- **Policy Updates**: [New or updated policies]
- **Training Needs**: [Identified gaps]

## Practical Exercises

### Exercise 1: Document a Past Incident
Using a real or hypothetical incident, complete a full response documentation form.

### Exercise 2: Timeline Creation
Create a detailed timeline of events for a sample incident showing:
- Initial detection
- Investigation activities
- Response actions
- Recovery steps

### Exercise 3: Evidence Log
Document evidence chain of custody including:
- What evidence was collected
- When it was collected
- By whom
- Where it's stored
- Access history

## Quality Checklist
- [ ] All required fields completed
- [ ] Timeline is chronologically accurate
- [ ] Evidence is properly documented
- [ ] Actions are clearly described
- [ ] Lessons learned are specific and actionable
- [ ] Grammar and spelling are correct
- [ ] Technical accuracy verified

## Completion Checklist
- [ ] Completed incident response report template
- [ ] Created detailed timeline
- [ ] Documented evidence chain of custody
- [ ] Reviewed for completeness and accuracy
- [ ] Submitted documentation package
